第一课：windows提权-快速查找exp

Posted on 2019-11-18

开始学习 https://github.com/Micropoor/Micro8

为什么要快速查找exp？

因为这样可以最安全的减少目标机的未知错误，以免影响业务。

如何查找？

大佬的命令是：

systeminfo>micropoor.txt&(for %i in ( KB977165 KB2160329 KB2503665 KB2592799

KB2707511 KB2829361 KB2850851 KB3000061 KB3045171 KB3077657 KB3079904

KB3134228 KB3143141 KB3141780 ) do @type micropoor.txt|@find /i "%i"|| @echo

%i you can fuck)&del /f /q /a micropoor.txt

分析一下：

第一部分：systeminfo>micropoor.txt

systeminfo： 可以显示关于计算机及其操作系统的详细配置信息，包括操作系统配置、安全信息、产品 ID 和硬件属性，如 RAM、磁盘空间和网卡。

> : 将当前命令输出以覆盖的方式重定向

总：把systeminfo命令执行后的结果以覆盖方式放到micropoor.txt文件中

第二部分：&(for %i in ( KB977165 KB2160329 KB2503665 KB2592799

KB2707511 KB2829361 KB2850851 KB3000061 KB3045171 KB3077657 KB3079904

KB3134228 KB3143141 KB3141780 ) do @type micropoor.txt

&： 顺序执行多条命令，而不管命令是否执行成功

（for %i in (…)）：一个循环，然后每次显示micropoor.txt文件

总：顺序执行后面的循环，并每次显示micropoor.txt内容

第三部分：|@find /i "%i"|| @echo %i you can fuck)

|：管道命令

|| ： 顺序执行多条命令，当碰到执行正确的命令后将不执行后面的命令

/i：不区分大小写

总：显示的文件内容通过管道命令，给到了find命令，并不区分大小写的寻找文件中存在的exp，如果该exp不存在，就会执行后面的echo命令，因为没有exp，证明可以被利用。反之，则不执行echo

第四部分：&del /f /q /a micropoor.txt

del命令下：

/f：强制删除只读文件。

/s：删除所有子目录中的指定的文件。

/q：安静模式，删除全局通配符时，不要求确认。

/a：根据文件属性删除文件

总：强制安静的删除micropoor.txt文件，不留痕迹。

命令的使用条件：

以上需要在可写目录执行。因为需要临时生成micrpoor.txt

一般实战中在类似 tmp目录等可写目录下执行：如C:\tmp>

补丁编号可以根据环境来增删
